# Plano: Galeria Lúdica da EBF — O Mapa das Memórias

## Contexto

A EBF 2026 (Escola Bíblica de Férias) aconteceu em 18/07/2026. A pasta `EBFIMG/` contém **47 fotos JPEG** + **10 vídeos MP4** (~5.9MB imagens, ~105MB vídeos). Todos nomeados pelo WhatsApp (`WhatsApp Image YYYY-MM-DD at HH.MM.SS[+digits].jpeg`), todos de 18/07/2026, sem subpastas ou categorias.

O objetivo é criar uma galeria lúdica ("O Mapa das Memórias") integrada à página `/ebf` existente, com descoberta automática de imagens, processamento incremental, mapa interativo com caça ao tesouro, galeria convencional, lightbox, destaques e encerramento cristão.

## Arquitetura

### Fluxo de dados

```
EBFIMG/*.jpeg  →  scripts/process-ebf-gallery.mjs (sharp)  →  public/images/ebf-gallery/*.webp
                                                                  ↓
                                                    src/data/ebf-gallery.generated.json
                                                                  ↓
                                                        EbfGallery.tsx (React)
```

### Dependência nova

- **`sharp`** (^0.33.0) — devDependency, processamento de imagens no Node.js

### Diretório de imagens derivadas

`public/images/ebf-gallery/` — **gitignorado** (artefato de build)

Cada imagem gera 3 arquivos WebP:
- `thumb_<id>.webp` (400px wide, ~15–30KB)
- `optimized_<id>.webp` (1200px wide, ~50–100KB)
- `placeholder_<id>.webp` (20px wide, LQIP, ~200B)

## Etapas de Implementação

### Etapa 1 — Script de processamento (`scripts/process-ebf-gallery.mjs`)

Script ESM que:

1. Lê `EBFIMG/gallery.config.json` (se existir)
2. Lê cache `.ebf-gallery-cache.json` (se existir)
3. Descobre arquivos `.jpeg`/`.jpg` em `EBFIMG/` (ignora `.mp4`)
4. Parseia timestamps do WhatsApp (incluindo sufixos decimais como `.445` = 44.5s)
5. Ordena cronologicamente (natural sort)
6. Para cada imagem: verifica cache (SHA-256) → se inalterada, pula
7. Gera com `sharp`: thumbnail (400px), otimizada (1200px), placeholder (20px blur)
8. Trata orientação EXIF
9. Distribui imagens em 5 etapas cronológicas equilibradas
10. Gera `src/data/ebf-gallery.generated.json`
11. Atualiza cache

**Distribuição em etapas** (cronológica, blocos equilibrados):
- Etapa 1 "Início da Jornada": ~10 imagens (início da faixa 1 + início da faixa 2)
- Etapa 2 "Primeiras Pistas": ~9 imagens (meio da faixa 2)
- Etapa 3 "Pelo Caminho": ~9 imagens (fim da faixa 2 + início da faixa 3)
- Etapa 4 "Grandes Descobertas": ~10 imagens (meio da faixa 3)
- Etapa 5 "O Tesouro Encontrado": ~9 imagens (fim da faixa 3)

### Etapa 2 — Tipos e utilitários

**`src/components/ebf/ebf-gallery.types.ts`** (ou no utils):
```ts
interface GalleryImage { id, src, thumb, placeholder, width, height, original, timestamp }
interface GalleryStage { id, name, clue, icon, images: GalleryImage[] }
interface GalleryManifest { generated, totalImages, stages: GalleryStage[], highlights: string[] }
```

**`src/lib/ebf-gallery-utils.ts`**:
- `parseWhatsAppTimestamp(filename)` → Date
- `naturalSort(a, b)` → number
- `distributeToStages(images, stageCount)` → GalleryStage[]

### Etapa 3 — EbfGallery (orquestrador)

Carrega o manifesto via import estático. Gereciencia:
- `selectedStage: number | null` (null = mapa, number = etapa aberta)
- `lightboxIndex: number | null` (null = fechado)
- Renderiza: MapaDasMemorias → GalleryGrid → Highlights → EbfClosing
- EbfLightbox como portal

### Etapa 4 — MapaDasMemorias

Baseado no padrão `MapaPeregrinacao.tsx`:
- Desktop: timeline horizontal com 5 marcadores conectados por trilha
- Mobile: timeline vertical
- Cada marcador: número, nome, ícone (lucide-react), contagem de fotos
- Estados: não visitado (cinza), visitado (dourado/amber), atual (brilho)
- Ao clicar: destaque o marcador, transição curta, carrega fotos da etapa
- "Voltar ao mapa" para retornar
- Caça ao tesouro: cada etapa esconde 1 pista (ícone de tesouro) que aparece após visitar
- Progresso em `localStorage` (`ebf-treasure-progress`)
- `prefers-reduced-motion`: desabilita animações

**Ícones das etapas** (lucide-react):
1. `Map` — Início da Jornada
2. `Binoculars` — Primeiras Pistas
3. `Footprints` — Pelo Caminho
4. `Gem` — Grandes Descobertas
5. `Trophy` — O Tesouro Encontrado

**Pistas (clues):**
1. "🔑 Uma chave antiga brilha no início do caminho..."
2. "🧭 A bússola aponta para a próxima descoberta..."
3. "🪙 Moedas douradas marcam o caminho percorrido..."
4. "💎 Uma pedra preciosa aguarda ser encontrada..."
5. "🏆 O maior tesouro estava no coração de cada criança!"

### Etapa 5 — GalleryGrid

Grade convencional com:
- Filtro por etapa (select)
- Ordenação cronológica (padrão)
- Paginação: 20 imagens por vez ("Carregar mais")
- Grid responsivo: 1 col mobile → 2 cols sm → 3 cols md → 4 cols lg
- Lazy loading via `loading="lazy"` + placeholder LQIP
- Dimensões explícitas (`width`/`height`) para prevenir layout shift
- Click → abre lightbox

### Etapa 6 — EbfLightbox

Baseado no padrão `VideoModal.tsx`:
- Portal via `createPortal(document.body)`
- `fixed inset-0 z-[100]` com backdrop blur
- Imagem centralizada, controles prev/next
- Contador "3 / 47"
- Badge com nome da etapa
- Botão download (quando permitido)
- Compartilhamento nativo (Web Share API)
- Teclado: ←/→ setas, Escape fecha
- Gestos: swipe esquerda/direita no mobile
- Foco: trap no lightbox, restore no fechamento
- Body scroll lock quando aberto
- Preload de prev/next

### Etapa 7 — Highlights

"Tesouros encontrados":
- 6–8 fotos destacadas (do manifesto, ou auto-selecionadas: 1–2 por etapa)
- Cards grandes com hover zoom
- Fallback: oculta seção se 0 destaques

## Acessibilidade

- HTML semântico (`<section>`, `<nav>`, `aria-labelledby`)
- `role="dialog"` + `aria-modal="true"` no lightbox
- `aria-label` em todos os botões interativos
- Foco visível (`focus-visible:ring-2`)
- Navegação por teclado (Tab, Enter, Escape, setas)
- `prefers-reduced-motion`: desabilita animações e transições
- Alt text genérico: `"Foto da EBF 2026 — {etapa} — {hora}"`
- Contraste: emerald-950 on #fff8dc → WCAG AA
- Áreas de toque ≥ 44×44px
- Nenhuma info transmitida exclusivamente por cor

## Privacidade

- Sem reconhecimento facial
- Sem identificação de crianças
- Sem metadados EXIF no navegador
- Sem GPS nas versões públicas
- Sem uploads por visitantes
- Sem comentários ou curtidas
- Download controlável por foto
- Avaliar `noindex` na galeria (verificar impacto na página EBF)

## Desempenho

- WebP via sharp (60–80% menor que JPEG)
- Thumbnails + LQIP = sem layout shift
- Lazy loading = ~10 imagens carregam no primeiro acesso
- Paginação (20 por vez) = carregamento incremental
- Processamento incremental = cache evita reprocessamento
- Total estimado: ~5MB imagens (lazy) + ~15KB JS + ~3KB manifesto
