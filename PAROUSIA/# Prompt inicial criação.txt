# Prompt final para agente de codificação — Hotsite da série “Da Ascensão à Parousia”

Você é um agente de codificação responsável por criar um hotsite responsivo para uma série de mensagens de uma igreja local.

O site principal da igreja já existe, é uma landing page simples, está hospedado no Netlify e está conectado a um repositório no GitHub. Não há CMS atualmente.

O objetivo é criar um hotsite estático, bonito, leve, responsivo, fácil de atualizar e anexável ao site da igreja.

---

# 1. Objetivo do projeto

Criar um hotsite para a série de mensagens:

**Da Ascensão à Parousia — O Livro da Longa Peregrinação**

Subtítulo:

**A Igreja entre o Reino inaugurado e a Cidade que vem.**

O hotsite deve funcionar como:

1. Guia pastoral para a membresia acompanhar a série;
2. Arquivo vivo com mensagens já pregadas, links, PDFs e materiais de apoio;
3. Ferramenta de divulgação para que membros compartilhem a série com convidados.

O hotsite será versionado no GitHub e publicado automaticamente pelo Netlify a cada commit na branch principal.

---

# 2. Conceito teológico e editorial

A série parte da ideia de que a Igreja vive entre a ascensão de Cristo e sua volta gloriosa. Cristo reina, mas a consumação ainda é aguardada. Enquanto isso, a Igreja caminha em missão, perseverança e esperança.

Preserve o seguinte princípio editorial:

**Atos narra a caminhada.**
**As Epístolas interpretam a caminhada.**
**O Apocalipse revela o destino da caminhada.**

O arco geral da série é:

**Ascensão → Pentecostes → Peregrinação → Deserto → Missão → Perseverança → Parousia → Nova Jerusalém**

Resumo teológico-poético da série:

**Cristo reina.**
**O Espírito conduz.**
**A Igreja caminha.**
**O deserto prova.**
**A missão se alarga.**
**A promessa demora.**
**O Cordeiro vence.**
**A estrada termina em casa.**

A frase conceitual que deve orientar o tom visual e textual é:

**Esperar é caminhar.**

---

# 3. Direção visual

O hotsite deve usar como inspiração a arte padrão da série já criada.

Características visuais desejadas:

* estética minimalista;
* atmosfera bíblica, sóbria e contemplativa;
* sensação de peregrinação;
* horizonte, caminho, estrada, deserto ou luz distante;
* referência sutil à cidade futura/Nova Jerusalém;
* bastante espaço negativo;
* boa legibilidade;
* visual adequado para igreja reformada/conservadora;
* evitar excesso de elementos decorativos;
* evitar aparência genérica de evento gospel moderno;
* evitar cores excessivamente vibrantes.

Sugestão de paleta:

* fundo escuro, grafite, preto ou azul profundo;
* tons terrosos, areia, bege ou dourado envelhecido;
* branco ou off-white para textos;
* acentos dourados discretos.

Tipografia sugerida:

* títulos com fonte serifada elegante ou display clássica;
* textos com fonte sem serifa limpa e legível;
* boa hierarquia entre título, subtítulo, corpo e botões.

A arte-base da série deve poder ser usada como imagem de fundo, imagem Open Graph, fallback visual dos cards, hero, materiais de divulgação e thumbnails.

```txt

↓
Salvar commit
↓
Netlify detecta a alteração
↓
```

1. Adicionar o PDF do sermão na pasta correta, se houver;

3. Inserir o ID ou link do vídeo do YouTube no arquivo `sermoes.json`;
4. Salvar o commit;
5. Aguardar o deploy automático do Netlify.

Não implementar banco de dados, login administrativo ou painel complexo neste primeiro momento.

# 9. Status automático dos sermões

Implementar uma função para calcular automaticamente o status do sermão com base na data e na existência de links.

Regras:

1. Se a data do sermão ainda não chegou, exibir:

**Em breve**

2. Se a data já passou e não existe `youtubeId`, `youtubeUrl`, `audioUrl` nem `pdfUrl`, exibir:

**Pregado — materiais em breve**

3. Se existe `youtubeId`, `youtubeUrl`, `audioUrl` ou `pdfUrl`, exibir:

**Disponível**

4. Se houver `statusManual`, ele pode sobrescrever o status automático.

Sugestão de função:

```ts
export function getSermonStatus(sermao: Sermon) {
  if (sermao.statusManual) {
    return sermão.statusManual;
  }

  const hoje = new Date();
  const dataSermao = new Date(sermao.data);

  const temMaterial =
    sermão.youtubeId ||
    sermão.youtubeUrl ||
    sermão.audioUrl ||
    sermão.pdfUrl;

  if (dataSermao > hoje) {
    return "em_breve";
  }

  if (!temMaterial) {
    return "pregado_materiais_em_breve";
  }

  return "disponivel";
}
```

Corrigir nomes de variáveis conforme o padrão do projeto.

Criar também um componente visual para status:

```txt
StatusBadge.tsx
```

Mapeamento sugerido:

```txt
em_breve → Em breve
pregado_materiais_em_breve → Pregado — materiais em breve
disponivel → Disponível
```

# 11. YouTube

Permitir duas formas de cadastro:

```json
"youtubeId": "ABC123XYZ"
```

ou:

```json
"youtubeUrl": "https://www.youtube.com/watch?v=ABC123XYZ"
```

Criar utilitários para:

* gerar link público do YouTube;
* gerar embed do YouTube;
* gerar thumbnail do YouTube, se possível;
* exibir botão para assistir no YouTube;
* incorporar o vídeo na seção “Mensagens já pregadas”, quando houver `youtubeId`.

Exemplo:

```ts
export function getYoutubeWatchUrl(youtubeId?: string, youtubeUrl?: string) {
  if (youtubeUrl) return youtubeUrl;
  if (youtubeId) return `https://www.youtube.com/watch?v=${youtubeId}`;
  return "";
}

export function getYoutubeEmbedUrl(youtubeId?: string) {
  if (!youtubeId) return "";
  return `https://www.youtube.com/embed/${youtubeId}`;
}

export function getYoutubeThumbnailUrl(youtubeId?: string) {
  if (!youtubeId) return "";
  return `https://img.youtube.com/vi/${youtubeId}/hqdefault.jpg`;
}
```

* tratar links ausentes;
